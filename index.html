Frequently Asked Questions

the difference between var, let, and const
  • var: Function-scoped, hoisted, and can be re-declared or re-assigned.
  • let: Block-scoped, not initialized during hoisting, and can be re-assigned.
  • const: Block-scoped, not reassignable, but allows changes to object properties or array elements.
the difference between map(), forEach(), and filter()
  • map(): Creates a new array by transforming every element in the original array.
  • forEach(): Iterates over elements in an array, performing a function on each without returning a new array.
  • filter():Creates a new array containing only the elements that meet a specified condition.

how JavaScript Promises work

Promises in JavaScript are like a placeholder for the result of an asynchronous operation (something that takes time to complete, like fetching data). They start in a "pending" state and move to "fulfilled" (success) or "rejected" (failure). You can use .then() to handle success, .catch() for errors

how closures work in JavaScript

Closures in JavaScript happen when a function can access variables from its outer scope even after that scope has finished executing. They work because inner functions "remember" the environment they were created in. This allows you to keep certain variables private and use them later.
